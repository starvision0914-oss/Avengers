서버 IP: **192.168.45.100** (Avengers 백엔드/프론트/DB/Redis/Samba 모두 동일 호스트).
2026-06-12 사용자가 "서버를 192.168.45.100으로 만들었어"라고 확정 → 네트워크 대역 192.168.45.0/24 로 이전.

**IP 변경 이력:**
- 192.168.2.16 — 폐기값. 2026-05-22 이전 시도했으나 실제 적용 안 됨. 어디서 발견되든 사용 금지.
- 192.168.1.16 — 직전값. 2026-06-04 라이브 확인까지 유효했음.
- 192.168.45.100 — **현재값**. 2026-06-12 사용자 확정.

**Why:** 사무실/네트워크 대역이 반복적으로 바뀐다. 과거 192.168.2.16 으로 옮긴다 했다가 실제로는 적용 안 되고 192.168.1.16 으로 환원된 전례가 있어, 코드/문서에 박힌 하드코딩 IP가 Samba/CORS/SMS 셋업을 반복적으로 망가뜨렸다. 이번 192.168.45.100 도 사용자 구두 확정값이므로 실서버 라이브 점검(ip route / ping)으로 한 번 더 확인하는 것이 안전하다.

**How to apply:**
- Samba 공유 경로: `\\192.168.45.100\PUBLIC`
- 웹 UI: http://192.168.45.100:5173 / API: http://192.168.45.100:8010
- 기존 파일에서 192.168.1.16 또는 192.168.2.16 을 발견하면 192.168.45.100 으로 갱신한다.
- 대역이 또 바뀔 수 있으니, 사용자 대면 안내는 가급적 접속주소 자동감지(window.location.hostname)를 우선한다.
