실데이터(구글시트 등)에 자동 업로드/덮어쓰기 기능을 만들 때, **올릴 내용의 양식을 먼저 확정·검증한 뒤** 실제 대상에 쓴다.

**Why**: 2026-06-14 11번가 광고비 구글시트 작업에서, 원본 스크립트가 올리던 보고서 형식을 확인 안 하고 "DB크롤러 보고서 = 시트가 원하는 것"이라 가정해 **상품별/키워드별(수만행)** 을 올림 → 시트 오염 + 구글 1천만 셀 한도 초과. 실제 정답은 **기간별 일자별 27컬럼(~30행)** 이었음.

**How to apply**:
- 원본/기존 양식을 먼저 확보(안 건드린 기존 워크시트 헤더 읽기, 원본 다운로드 CSV, 또는 사용자에게 헤더 요청).
- 막히면 추측 남발 말고(여기선 reportScope 20개 다 400 낭비) **실제 호출 URL/네트워크를 캡처**하거나 사용자에게 페이지 주소를 받는다.
- 실제 대상 덮어쓰기 전 **1계정·소량으로 형식 검증** 후 확대. 되돌리기 어려운 외부 반영은 특히.
